## Architektur-Notizen

### Unified System Struktur
```
UnifiedCard (game, name, type, etc.)
    └── UnifiedPrinting (collector_number, set, rarity, finish, image_url)
            └── UnifiedInventory (user_id, lot_id, condition, language, quantity, extra)
```

### CardMatcherService Pattern
- Jedes Spiel hat eigenen Service: `FabCardMatcherService`, `MtgCardMatcherService`, etc.
- Alle implementieren `CardMatcherInterface`
- `findMatch()` gibt jetzt UnifiedPrinting zurück (nicht mehr game-spezifische Printings)

### Scanner Flow
1. Kamera/Suche → `ScannerController::recognize()` oder `search()`
2. CardMatcherService findet UnifiedPrinting
3. `ScannerController::confirm()` erstellt UnifiedInventory
4. Frontend erhält `lotInventory` via Props (kein optimistisches Update mehr)
